## 1. Monday — Diario di Bordo (behind-the-automation)

**Objective**: Build trust, show systems work in reality, differentiate from theory-only creators
**Primary emotion**: Curiosity + vulnerability
**Character target**: 500-800 (micro to standard)
**Format**: Dual publication — LinkedIn post + blog post (optional, link in first comment)

### Concept
"Diario di Bordo" = Weekly diary entry from Claude's perspective (or from Giovanni reflecting on how the automation week went). Shows:
- Real numbers (impression crashes, engagement bugs, unexpected wins)
- The human learning moment (what broke, why, how I fixed it)
- Vulnerability (the mistake I should have seen)
- Concrete outcome (specific code line or change)

### Hook pattern (always time + scene)
- "Lunedì mattina. La Skill di engagement si è bloccata alle 8:03."
- "La dashboard ha mostrato qualcosa che non mi aspettavo."
- "Ho iniziato la settimana con 14 automazioni in produzione. Una non era pronta."

### Body structure
1. **What happened** (specific event, timestamp, number)
2. **What surprised me** (the data revealed, the assumption that was wrong)
3. **The fix** (3 lines of code, 1 new rule, a pivot)
4. **Parenthetical reflection** (humanization, the irony, what I should have seen)
5. **The result** (outcome: skill didn't crash again, engagement improved, process refined)

### Closing options
- Signature: "Il sistema funziona. Tu fallo partire."
- Reflective question: "Quando l'avete sperimentato per la prima volta, che cosa avete notato?"
- Mic-drop: "E il valore è in quello."

### Real example (Week 3)
```
Lunedì mattina. La Skill di engagement si è bloccata alle 8:03.

Non un crash. Un overflow della context window.
Il sistema ha provato a processare 47 notifiche in un colpo solo.

Il mio primo istinto: aumentare la finestra.
Poi ho guardato i dati. 37 di quelle 47 erano reazioni "ottimo post".

Il fix non era tecnico. Era strategico:
filtrare prima di processare, non dopo.

(onestamente, avrei dovuto prevederlo)

3 righe di codice. La Skill non si è più bloccata.

Il sistema funziona. Tu fallo partire.

#DiarioDiBordo #AIAutomation #Claude
```

**First comment (20 min after):**
```
Diario completo con tutti i dati della settimana: giovanniliguori.it/blog/diario-di-bordo-settimana-03
```

## 2. Tuesday — Tool/Workflow (full-stack)

**Objective**: Utility, generosity, practical implementation
**Primary emotion**: Pragmatism + generosity
**Character target**: 800-1200
**Focus**: Full-stack (Claude Cowork/Code + Python + Google Cloud)

### Hook pattern (utility + ecosystem)
- "Il setup che mi fa risparmiare X ore/settimana."
- "Layer 1... Layer 2... Layer 3... [ecosystem breakdown]"
- "3 tool che parlano uno con l'altro. Come funziona."

### Body structure
1. **Problem before** (what was manual)
2. **The layers** (specific tools, how they connect)
3. **The orchestration** (timing, cascade, what triggers what)
4. **Proof** (measurable: hours saved, automation deployed)
5. **The bottleneck** (parenthetical insight on what's actually hard)

### Real example (Week 3)
```
Il setup che mi fa risparmiare 6+ ore/settimana su LinkedIn.

Non è un tool. È un ecosistema.

Layer 1: Claude Cowork orchestra tutto. Cron task, sub-agenti, pipeline.
Layer 2: Python gestisce le API e il data processing.
Layer 3: Google Cloud Run fa girare i job in background.

Il bello è che ogni layer parla con gli altri.
Il cron task delle 8:00 pubblica il post.
Alle 8:20 un sub-agente aggiunge il primo commento.
Alle 9:00 parte la sessione di engagement.

Tutto automatico. Tutto tracciato. Tutto in produzione.

(spoiler: il collo di bottiglia non era il codice. Era decidere cosa automatizzare e cosa no.)

Il sistema funziona. Tu fallo partire.

#Claude #Automation #Productivity #AIWorkflow
```

**First comment (same day, 20-30 min after):**
```
La parte più sottovalutata? Decidere cosa NON automatizzare. Le risposte ai DM, per esempio, restano manuali. Il contesto umano lì è insostituibile.
```

## 3. Wednesday — Hot Take (Opinion)

**Objective**: Engagement, comments, moral clarity
**Primary emotion**: Indignation (controlled) + certainty
**Character target**: 600-900
**Critical rule**: Must take a clear position. Not neutral analysis.

### Hook pattern (observation + judgment)
- "Ho visto l'ennesimo [thing]. È sbagliato. E vi spiego perché."
- "Unpopular opinion: [statement that challenges consensus]."
- "Mi chiedo se non stiamo confondendo [X] con [Y]."

### Body structure
1. **The observation** (what you saw)
2. **Clear position** (this is wrong/this matters/this worries me)
3. **The evidence** (1-2 data points, examples, or reasoning)
4. **Implication** (what happens if this continues)
5. **The real question** (dimensional reframe or "real question" pattern)

### Real example (Week 3)
```
Ho visto l'ennesimo "corso AI" a €997 che insegna a scrivere prompt.

È sbagliato. E vi spiego perché.

Un prompt non è una skill. È un input.
Scrivere un buon prompt senza capire context window, temperature, system instructions e tool use è come scrivere una query SQL senza sapere cos'è un database.

Il mercato si è polarizzato: da un lato chi vende "il prompt magico", dall'altro chi costruisce sistemi.

Mi chiedo se tra 12 mesi qualcuno si ricorderà dei prompt engineer.

Chi già costruisce pipeline ha un vantaggio enorme su chi aspetta che diventi best practice.

#AI #Opinion #Freelance
```

**First comment (Wednesday-specific):**
```
La domanda vera: quanti di quei corsi mostrano un sistema in produzione? Non slide. Non demo. Produzione.
```

## 4. Thursday — Case Study (Proof)

**Objective**: Social proof, credibility, measurable results
**Primary emotion**: Pride (controlled) + specific details
**Character target**: 1000-1300 (longest post of the week)
**Rule**: Micro-anecdote with sensory detail, not abstract metrics

### Hook pattern (specific quote or before/after)
- "Giovedì scorso un [client type] mi ha scritto: '[specific quote]'."
- "Before: [pain]. After: [sensory detail showing success]. The number: [metric]."

### Body structure
1. **The situation** (who, what problem, context, pain)
2. **The approach** (what you built, specific tools, stack)
3. **The result** (numbers: hours/cost/revenue + the sensory detail that proves it worked)
4. **The insight** (what this teaches about broader pattern)
5. **The conclusion** (measured, not hype)

### Real example (Week 3)
```
Giovedì scorso un consulente mi ha scritto: "Per la prima volta in 6 mesi ho pranzato senza il laptop."

Non è una frase motivazionale. È il risultato di 3 automazioni.

Il suo problema: 40+ ore/mese spese a fare report manuali per 12 clienti. Copy-paste da Google Analytics, formattazione in Slides, invio via email. Ogni mese, 12 volte.

Il sistema che abbiamo costruito:
→ Claude legge i dati via API (GA4 + Search Console)
→ Python processa e genera il report in PDF
→ Cloud Run lo schedula il primo del mese
→ Resend lo invia al cliente con un messaggio personalizzato

Tempo totale dopo il setup: 0 ore/mese.
Il setup: 2 giorni.
Il ROI: dal primo mese.

Il dettaglio che mi ha colpito: non ha detto "risparmio tempo".
Ha detto "pranzo senza laptop".

Ecco. Il valore non si misura solo in ore. Si misura in qualità della giornata.

Il sistema funziona. Tu fallo partire.

#CaseStudy #AIAutomation #B2B #Claude
```

```

## 5. Friday — How-To (Educational)

**Objective**: Maximum value, saves, reach (best day: 35.9 avg impressions)
**Primary emotion**: Patience + thoroughness
**Character target**: 1000-1300
**Format**: Step-by-step with the non-obvious insight

### Hook pattern (outcome + timeframe)
- "Come [achieve result] in [timeframe]. Step by step."
- "Il [tool] setup che nessuno spiega bene."
- "State usando [tool] male. Ecco il modo giusto."

### Body structure
1. **Outcome** (what you'll achieve)
2. **Steps** (3-5 specific steps, each 1-2 sentences, with code/config if needed)
3. **The trick** (the non-obvious part most people miss)
4. **Where to go deeper** (blog post, docs, or next step)
5. **Closing validation** ("Funziona? Funziona.")

### Real example (Week 3)
```
Come creare la tua prima Skill Claude in 15 minuti. Step by step.

Una Skill è un file SKILL.md che dice a Claude come comportarsi in un contesto specifico. Non è un prompt. È un'istruzione persistente.

Step 1: Crea una cartella nella directory del tuo progetto.
Esempio: /skills/email-writer/

Step 2: Crea SKILL.md con questa struttura:
→ name: il nome della skill
→ description: quando deve attivarsi
→ Il corpo: le istruzioni operative

Step 3: Nel corpo, sii specifico.
Non "scrivi bene". Ma "usa frasi di max 2 righe, apri con un dato, chiudi con una domanda. Mai usare 'cordiali saluti'."

Step 4: Testa con un prompt reale.
Scrivi qualcosa che attiverà la skill e verifica che il risultato rispetti le regole.

Step 5: Itera.
La prima versione non sarà perfetta. La terza sì.

Lo so, sembra banale. Ma la differenza tra chi usa Claude come chatbot e chi lo usa come sistema è esattamente questa: le Skill.

Funziona? Funziona.

Il sistema funziona. Tu fallo partire.

#Claude #HowTo #Skills #Tutorial
```

```

## 6. Saturday — Storytelling

**Objective**: Human connection, personal trust
**Primary emotion**: Tenderness + reflection
**Character target**: 500-900
**Format**: Personal scene with a lesson

### Hook pattern (time reference + personal scene)
- "[Time reference]. [Personal scene]."
- "Due anni fa [specific situation]."
- "Mi capita di pensare spesso a [moment]."

### Body structure
1. **The scene** (specific moment, sensory details, emotion)
2. **The realization** (what it taught you)
3. **The broader lesson** (applicable to reader)

### Real example (Week 3)
```
Due anni fa editavo video 12 ore al giorno.

Oggi non tocco Premiere da 8 mesi.
Non perché non mi piaccia. Perché ho trovato una leva più grande.

A un certo punto ho capito che il montaggio migliore che potessi fare non era su una timeline.
Era su un workflow.

E niente. Ogni tanto ci penso.

#Storytelling #CareerChange #AI
```

**First comment (Saturday-specific: insight or parallel):**
```
Il parallelo che nessuno vede: editare un video e costruire un'automazione richiedono lo stesso muscolo. Ritmo, timing, attenzione al dettaglio. Cambia solo il medium.
```

## 7. Sunday — Soft CTA (Conversion)

**Objective**: Product awareness, conversion (only day "venditore" tone is allowed)
**Primary emotion**: Confidence + directness
**Character target**: 500-800
**Format**: Problem from DM → Your solution

### Hook pattern (question from DM, or reader problem)
- '"[Real DM question]."'
- "Me lo chiedono in DM almeno X volte a settimana."
- "Il [resource] che mi sarebbe servito quando ho iniziato."

### Body structure
1. **The problem/question** (something your audience recognizes)
2. **Your solution** (the product, with specifics: pages, modules, proof)
3. **The proof** (1 testimonial or result)
4. **The price as value** (€X = ROI timing)

## 8. Micro-Post Template

**Frequency**: At least 1 per week
**Length**: 200-400 chars (3-6 bare sentences)
**Structure**: One thought. No framework. No bullets. No list.

### Real examples
```
Ho passato 3 ore a debuggare una Skill che avrebbe dovuto funzionare al primo colpo.
Non ha funzionato.
Il problema non era il codice.
Era l'assunzione che ci avevo messo dentro.
```

```
Il miglior commento che ho ricevuto questa settimana non era "ottimo post!"
Era "non sono d'accordo perché [motivo specifico]."
Quello è il segnale.
```

```
21 automazioni.
Zero esistevano 8 mesi fa.
Ognuna ha sostituito un task che facevo a mano.
Il tempo è tornato. La qualità è salita.
E non ho assunto nessuno.
```

## 9. Character Count Targets & Rationale

| Giorno | Target | Min | Max | Rationale |
|--------|--------|-----|-----|-----------|
| **Mon** | 500-800 | 350 | 900 | Micro/standard. Vulnerability is concise. |
| **Tue** | 800-1200 | 700 | 1300 | Full-stack ecosystems need explanation. |
| **Wed** | 600-900 | 550 | 1000 | Hot takes punchy. Position + evidence + implication. |
| **Thu** | 1000-1300 | 900 | 1400 | Case studies are longest. Detail justifies length. |
| **Fri** | 1000-1300 | 900 | 1400 | How-tos need step detail. Best-day halo. |
| **Sat** | 500-900 | 400 | 1000 | Storytelling can be micro or standard. |
| **Sun** | 500-800 | 450 | 900 | CTA short. Problem + solution + price. |

**Weekly balance rule**: Never all the same length. Target 1 micro (200-400), 3-4 standard (600-1000), 1-2 long (1000-1300). Varied rhythm signals human writing, not AI template.
